## 图鉴暂停游戏功能

### 需求
打开图鉴或观看图鉴时暂停游戏，关闭图鉴窗口后游戏自动继续运行。

### 实现方案
1. 添加 `isPaused` 状态变量，标记游戏是否暂停
2. 在 `showCollection()` 函数中：
   - 清除计时器 `clearInterval(timerInterval)`
   - 设置 `isPaused = true`
3. 在 `hideCollection()` 函数中：
   - 重新启动计时器 `setInterval(updateTimer, 1000)`
   - 设置 `isPaused = false`
4. 在 `selectCard()` 函数中添加暂停检查，暂停时不能操作卡牌

### 关键代码
```javascript
// 打开图鉴时暂停
if (!gameOver && timerInterval) {
    clearInterval(timerInterval);
    timerInterval = null;
    isPaused = true;
}

// 关闭图鉴时恢复
if (!gameOver && isPaused && !timeOut) {
    timerInterval = setInterval(updateTimer, 1000);
    isPaused = false;
}

// 点击卡牌时检查
if (isPaused) return; // 暂停时不能操作
```

## 修复第一关逻辑并使用PSD图片显示

### 问题
1. 第一关卡牌生成逻辑有误，最后剩余的三张卡牌永远不一样，无法通关
2. 海星、章鱼和寄居蟹使用的是emoji而不是用户提供的PSD图片

### 解决方案
1. **修复卡牌生成逻辑**：
   - 原逻辑：`for (let i = 0; i < adjustedCardCount; i++) { cardTypes.push(LEVEL1_SPECIAL_FISH[i % 3].emoji); }`
   - 修复后：`const fishPerType = adjustedCardCount / 3;` 确保每种鱼数量相同
   - 例如18张卡：每种鱼6张，可以形成3组完整的消除

2. **使用PSD图片**：
   - 将Fish_0001.psd、Fish_0002.psd、Fish_0003.psd转换为small版本（65x85px）
   - 将PNG图片转换为Base64编码
   - 在`LEVEL1_SPECIAL_FISH`配置中添加`image`字段
   - 修改`renderCards()`和`renderSlot()`函数：
     - 第一关（level === 1）使用图片显示
     - 其他关卡继续使用emoji

### 关键代码
```javascript
// 第一关卡牌生成逻辑修复
if (level === 1) {
    const fishPerType = adjustedCardCount / 3;
    for (let fishIdx = 0; fishIdx < 3; fishIdx++) {
        for (let j = 0; j < fishPerType; j++) {
            cardTypes.push(LEVEL1_SPECIAL_FISH[fishIdx].emoji);
        }
    }
}

// 渲染卡牌时使用图片
if (level === 1) {
    const fish = LEVEL1_SPECIAL_FISH.find(f => f.emoji === card.type);
    if (fish && fish.image) {
        cardEl.innerHTML = `<img src="${fish.image}" style="width:100%;height:100%;object-fit:contain;" alt="${fish.name}">`;
    }
}
```
